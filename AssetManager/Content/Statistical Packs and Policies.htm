Statistical Packs and Policies

Statistical Packs are groups of Statistical Attributes configured to work together to mine historical data and compute at specified times. Statistical Attributes must be created from Asset Attributes or System Attributes before Statistical Packs and Statistical Policies can be configured.

To Create a Statistical Attribute for inclusion in a Statistical Pack

  1. Navigate to Admin Console > Asset Attributes or Admin Console > System Attributes.

  1. Select an Attribute.

If the selected attribute is one for which a Statistical Attribute cannot be created, the New Statistic button will appear grayed out. Statistical attributes can be used with any attribute that produces a numeric value, regardless of the asset type.

  1. Click New Statistic to create a statistical attribute from the selected attribute. The statistical computations supported for Statistical Packs are Maximum, Minimum, Median, Average, and Standard Deviation. For most Statistical Packs, all five Statistical Attributes will be created.

  1. Assign a name and optional description. Select the Statistic from the drop-down. Click Save Changes. The new Statistical Attribute appears in the Attributes tree.
  2. Click to select the parent Attribute again and click New Statistic. Repeat for the other desired statistics.

Statistical Attributes can be used as other Attributes.

To Create a Stat Pack

  1. Navigate to Admin Console > Data Schema > Statistical Pack and click New.
  2. From the drop-down menu, select the Source Attribute under which you created the Statistical Attributes to be grouped into a Pack.
  3. From the drop-down menus, select the statistical attributes you created.

  1. Set the historical record data window and update frequency.
  • Daily calculates at midnight local time
  • Weekly Sundays at midnight
  • Monthly on the first of the month at midnight

  1. Save Changes.
  2. Navigate to Admin Console > Data Schema > Statistical Policy and click New.
  3. Complete all required and any desired additional fields.
  • Name: Assign a name to the asset attribute. The name is used on all forms to prompt for the asset attribute data, so it is advised to use a clear, concise and intuitive name for the new attribute.
  • Description: If desired, input a detailed comment describing purpose or meaning of the asset attribute.
  • Filter Asset Type: Select an Asset Type to which this Policy applies. Policies apply to all sub-levels beneath the selected level.
  • Filter Location:Select a Location to which this Policy applies. Policies apply to all sub-levels beneath the selected level.
  • # Attribute: Select Attributes to restrict the Policy to only those attributes. All Attributes created in the system appear in the drop-down.
  • # Attribute Value: Select Attribute Values or Value Operators to restrict the Policy only to those values or operators.
  • Statistical Packs: Click the Ellipsis ... button to access the selection dialog. Move Statistical Packs from Available to Selected and click OK when finished.

  1. Click Save Changes.

Statistical Packs can be used as other Attributes.
