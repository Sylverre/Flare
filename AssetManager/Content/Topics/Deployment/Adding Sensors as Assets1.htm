Adding Sensors as Assets

In order to view data from a sensor in , the sensor tag must be assigned to an asset or as an asset assigned with a location.

To add a new sensor tag as an asset:

  1. Navigate to User Console > Tag Management > Manage Tags and right-click to select a tag from the far right column, Unassigned Tags.
  2. Click New Asset.
  3. Select the appropriate Asset Template or Asset Type. (Use the + to expand selections if required.)
  1. Complete all required and any desired additional fields.

  • Name: Assign a name to the sensor. The name is used on all forms to prompt for the data, so it is advised to use a clear, concise and intuitive name for the new attribute.Typically pre-pending a type designator (TempHum), then a location descriptor will suffice.
  • Asset Tag: Depending on the method used to get to this screen, this field may be pre-populated. If not, simply start typing the Tag ID of the tag and select the appropriate Tag ID from the dynamic list presented.
  • Assigned To: This field is used to tell whether the sensor tag is affixed to a summary location such as a rack.
  • Description: If desired, input a detailed comment describing the sensor tag asset.This can be any information that is helpful to you when monitoring the environment. For example, you could note that a temperature tag is installed on a rack door or on the cabinet itself.
  • Assigned Location: This field is used to tell where the tag will reside physically, and is designated using the Location hierarchy.
  • Expected Location (s): This is where an asset should be and is also designated using the Location Hierarchy that you have already configured. However, this field—Expected Location—is intended to be used with mobile assets (hospital equipment, shipping trucks, etc.), that should be one place but that are detected in another; it is very rarely used with sensor tag assets.
  • Lock Location: For non-mobile assets, such as environmental sensors, selecting this checkbox is important, as sensor tags should not be physically moved after deployment.
  • Temperature Sensor Application: This pull-down menu designates an intended use for the sensor tag asset, for example, on an IT Rack, and enables other fields, depending on the application selected, for example, the Airflow Position and the Rack Position menu fields will appear when the “application” selected is an IT Rack Sensor.
  • Airflow Position: This field designates if the sensor is in the front or the back of the cabinet or rack. Rack Intake Temperature corresponds to the front and Rack Exhaust Temperature corresponds to the back of the rack.
  • Rack Position: This field designates whether the sensor tag is at the Top, the Middle, or the Bottom of the rack door.
  1. Click Save.
