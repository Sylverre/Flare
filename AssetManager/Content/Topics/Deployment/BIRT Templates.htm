BIRT Templates

BIRT Templates: Business Intelligence and Reporting Tools (BIRT™) is an open-source report generation tool developed by the Eclipse™ Foundation. You can create reports in the BIRT Designer and then add them to where the report will be executed. The BIRT module enables a rich variety of features, such as custom formatting, charts, data grouping and logos. You can even include data from multiple reports and external data sources to create reports tailored to your specific needs.

For more information about BIRT templates, visit http://www.eclipse.org/birt

BIRT™ (Business Intelligence and Reporting Tools) is an open-source report generation tool developed by the Eclipse™ Foundation. To create report templates for use in , download the BIRT Designer, a tool that allows you to create reports, add graphical elements, adjust layout, and link to data sources. There are two main components of BIRT:

BIRT Designer: A graphical report-designing tool that allows you to create reports, add graphical elements, adjust layout, and link to data sources.

BIRT Engine: A software component that runs the reports created in the Designer.

Reports created in the BIRT Designer can be added to , where the reports will be executed. With BIRT, you can add a rich variety of features to reports such as custom formatting, charts, data grouping and logos. You can even include data sets from data sources other than , which can then be integrated with r data to create reports that integrate the various software systems you use.

Please note that in order to use BIRT within you must purchase and install an BIRT license key. For instructions on installing this license key, please refer to the License Key Configuration section of this document.

The following is an example of how to use the BIRT Designer. BIRT has many advanced features that enable you to create complex reports.

Integration with BIRT Reporting

If the BIRT License key has been installed by an administrator you will be able to create a BIRT Template for use in running a BIRT enabled report in Asset Manager. To do this perform the following steps:

1. Create a report in . This report will become the data source that BIRT uses to generate its report.

2. Run a sample of the report and export it as a BIRT Design or CSV file. Save the file in its own directory. If the export is a ZIP file, extract the files.

Creating a BIRT Project

3. Access and open the BIRT Designer and navigate to File > New > Project. Select “Report Project” in the BIRT Wizard.

4. Name the project and click Finish.

Creating a BIRT Report

5. To create a report, select the “Navigator” tab in Eclipse, then right-click your project and select New > Report. You will be prompted with a window asking you to name your report. Enter a name and click Next.

6. Next you will select your report type. There are several pre-defined choices with various options. For this example, just choose “Blank Report” and click Finish.

Connecting to Data within BIRT Designer

BIRT Designer uses “Data Sources” to connect to the data you would like to use in your report. There are many ways that BIRT can connect to data for report generation. Creating “data sources” like these teaches BIRT how to retrieve data from various sources to combine into the reports. For example:

  • You can connect to most DBMS systems using SQL queries with JDBC Data Sources.
  • Web Services and XML files can be accessed for report data.
  • Simple flat-file systems can be used as data sources.

7. In the Data Explorer tab or in the Outline tab, right-click Data Sources and choose New Data Source and create a new flat-file data source. The new data source MUST be named “RFCodeDataSource”. When prompted, use the directory where you stored your sample BIRT Design or .CSV file as the directory for the new data source.

A flat-file data source is a BIRT data source that references a file-system folder which contains one or more CSV files.

Once the flat-file data source is added to your BIRT report, each .csv file in the folder can be configured as a BIRT Data Set. Like all BIRT Data Sets, many things can be done to manipulate the data within BIRT, such as creating joins between data sets, using computed columns, aliasing, etc.

Specifying Data Sets

Data Sets are derived from Data Sources in BIRT. Each Data Set becomes it’s own table of data and can be displayed and manipulated.

8. In the Data Explorer tab or in the Outline tab, right-click Data Sets and choose New Data Set.

9. Click Next, then create the query: Select the CSV file that you created from the report generation and choose the columns that you would like to have in your BIRT Report. If a Data Source contains multiple CSV files, create a data set per CSV file. Click Finish and click OK to confirm the column selections.

Adding Data to a BIRT Report

10. To add the data from your data set to your report, select the Outline (or Data Explorer) tab in Eclipse and drag and drop the data set onto your report. This will create a table based on the data set.

A data set on the design area is a “table”. You can customize formatting for this table in the Table’s property section.

11. Click the Preview tab to view your report. To view the report in a different format, use the menus to select Run > View Report > As PDF (or as other format as desired).

Accessing BIRT Report XML Source

12. The XML Source tab in the BIRT Eclipse environment displays the report source code. The primary purpose of the BIRT Designer is to create this XML file. This XML File will be used to integrate with Asset Manager. The report XML file (.rptdesign) can be found inside your project in the Eclipse Workspace directory.
