Editing Your Configured Database

  1. Navigate to the Configuration>Database to access the database configuration that you created in the Getting Started steps.

Figure 48: Database Sub-Task

  1. Edit your Database Configuration Setting fields. These fields include the following:
  • Database Type: Type of database that you will be using with . You have a choice of either Microsoft SQL Server, PostgreSQL, Oracle 11g Server, or IBM DB2 Server.
  • Hostname: Enter your valid hostname or IP address of the database server.
  • Port: The port number you will use to connect to your database. In most cases the default value for connecting to Microsfot SQL Server will be port number 1433 (PostgreSQL = 5432, Oracle 11g = 1521, IBM DB2 = 50000). If you are unsure of your port number, contact your database administrator.
  • Database Name: The name that was assigned to your database.
  • Windows Authentication Mode Checkbox: Checked if your database requires Windows authentication. Please see your database administrator to determine this.
  • Username: Fill in the username that allows for full read/write privileges. If you have checked the Windows Authentication Mode Check box, you will not see this field because your database will require your Windows password and username.
  • Password: Enter the password associated with the username mentioned above.
  • Email Address: In addition to editing these fields, you can also enter/edit an email addresses to send the database alerts in the “Contact When Connection to Database Fails” section of the task pane.
  1. Once you have completed the form, click the Test button. If you have entered the information correctly and is able to connect to your database, you will receive a message indicating that the test was successful.
  1. Click the Save Changes button to save your edits for the configuration.
