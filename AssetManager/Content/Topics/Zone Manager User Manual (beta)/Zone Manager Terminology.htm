Zone Manager Terminology

Listed below are a list of Zone Manager terminology and concepts that are used in the Zone Manager software:

Reader:

  • reader such as the M200, D200, R200.
  • Zone Manager only supports IP connections to readers.
  • Each reader must be defined (created), then setup (IP address, plus additional parameters)

Tag:

  • active tag such as the M series, D series or R series tags. Each tag has a unique ID and a group code.

Tag Groups:

  • Are the list of tag group codes and corresponding tag treatment codes needed by Zone Manager to interpret tag events correctly.
  • All unique tag group codes must be properly configured in Zone Manager

Locations:

  • Define “places” or “areas” inside Zone Manager.
  • In its most basic form, a Location is merely a label name.

Location Hierarchies:

• Define which Location children parent relationships

• This allows Zone Manager to have a Location path such as “Tag 1 is in Rack 5 which is in Row 13 in Data

Center A”.

Location Attributes:

  • Locations can also be defined with additional user defined attributes such as a city name, or a rack type, etc..
  • Location Attributes can be defined at any level in the location hierarchy. All children inherit the attributes of their parents.

Example:

◊ A location called “Data Center A” is created as the top level location (no parent) with a user defined attribute of “city=Austin”

◊ A location called “Row 1” is created with “Data Center A” as its parent with a user defined attribute of “racktype=IBM”

◊ A location called “Rack 3” is created with “Row 1” as its parent with no user defined attributes

◊ A query of all tags in location “Rack 3” would return a list of those tags and all attributes of those tags.

The response would look like the following:

RFCRCK00000012 locationzone=Rack 3, city=Austin, racktype=IBM

Rule Types:

  • Rule algorithms that are available for use to build location rules
  • Currently there are three types of rules included with Zone Manager:

◊ SimpleSSIRule - which thresholds on the strongest SSI reported from a set of one or more radio channels associated with the rule):

◊ AverageSSIRule - which thresholds on the average SSI reported from a set of one or more radio channels associated with the rule, and requires ALL listed channels to be able to read the tag):

◊ IRLocatorRule - which compares the ‘irlocator’ attribute of tags supporting it with a specific value, and can optionally require that one of a set of channels be able to read the tag):

Rules:

  • Logic used by Zone Manager that when parameters are met, determines which location (zone) a tag is located in.
  • If no rules match, then the tag location is considered “unknown”
