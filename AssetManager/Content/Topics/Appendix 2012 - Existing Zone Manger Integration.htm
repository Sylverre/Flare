Appendix 2012 - Existing Zone Manger Integration

This section will describe the process of importing an existing remote Zone Manager configuration for use with . This process is done through an exportation of the four Zone Manager configuration files (readers. csv, groups.csv, locations.csv, and rules.csv) from the existing remote Zone Manager, and then importing these files into . The purpose of performing this action is to migrate reader, tag group, location and rule information that you have already configured with your existing remote Zone Manager into .

This step is for users who already have an existing configured Zone Manager system, if you are installing a remote Zone Manager for the first time, you will not need to perform these steps. For information on configuring remote Zone Managers for use with , please refer to steps number 10-12 of the Getting Started section of this document.

Exporting Configuration Files from Remote Zone Manager

The following is a list of instructions to export your existing remote Zone Manager configuration files:

1. Access the system where you are currently running your Zone Manager. In order to ensure that files are updated with the most recent configuration operations you have performed with your existing Zone Manager, you must restart the Zone Manager service. This will automatically update the Zone Manager configuration files. To do this please perform the following:

a. Navigate to Start > Control Panel > Administrative Tools > Services.

b. Locate and select the RF Code Zone Manager service. Click Restart to restart the service (This will update the existing Zone Manager configuration files).

2. Navigate to the C:\Program Files\RF Code\Zone Manager\zonemgr.datadir (Windows users) or usr\share\ rfcode\zonemanager\jetty\zonemgr.datadir (Linux users) directory. Here you will find four files: readers. csv, groups.csv, locations.csv, and rules.csv. Save a copy of these files to the system that will be running the console.

3. Start the console and import the existing remote Zone Manager configuration files into . Do this by navigating to Configuration > Import Configuration on the Administrative Console. The import configuration screen will appear.

4. Browse to the existing Zone Manager configuration files (groups.csv, readers.csv, locations.csv, rules.csv) that you just copied to your system and click Upload. Once the files have uploaded a message box will appear indicating that you have imported the files successfully.

This completes the existing Zone Manager integration process. The imported tag groups, readers, locations, and rules will appear in the appropriate configuration panels in the Administrative Console.

Please note that the existing Zone Manager configurations are now assigned to the local Zone Manager.

This means that you must turn off your existing Remote Zone Managers if using M200 Readers since RF Code M200 Readers only support a single simultaneous connection.
