Allocating Memory to

By default the amount of memory that is allocated to the software is configured for relatively small deployments. For many installations this amount of memory is sufficient, however if the number of assets exceeds roughly 5,000 or the calculated field function is heavily used, additional memory resource may need to be allocated to maintain good performance characteristics. Adding physical memory to the server that runs may help if swapping is experienced but to fully utilize the free memory, the software must be defined to allow to use more memory than the default.

To change the amount of memory being used by , edit system.properties file and change the value of the memory directive. The following example sets the memory footprint to 2 GB or 2048 MB of memory:

wrapper.java.maxmemory=2048

The maximum amount of memory that can be allocated to on a 32bit operating system is 4096mb. Depending on the version of Microsoft Windows you are running, the operating system may limit the amount of physical memory that can be addressed. The use of 64bit versions of Windows Operating System is encouraged when more than 2GB of memory is needed.
