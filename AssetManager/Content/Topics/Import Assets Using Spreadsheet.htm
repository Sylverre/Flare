Import Assets Using Spreadsheet

All of your assets and sensors can be manually added to the database, but it is more efficient to upload this information from a populated spreadsheet.

  • must be configured before Assets can be imported.
  • At least one Asset must be created within ..
  • File must be created in and exported from , then populated and imported into system.
  • Every asset must have a unique name. Duplicate assets will not be imported.
  • There is no limit on the number of asset types that can be created within .
  • Assets can be imported all at once or in smaller sets.
  1. Navigate to User Console > Assets > Manage Assets. If no assets have been created, click New to create at least one asset manually in system.

 

If importing a set of assets all of a single type, create an asset of that type.

  1. Click Export. Select Advanced Export and CSV.
  2. Open file in spreadsheet program.
  3. Highlight all cells (Ctrl-A) and then double-click the line between the first two columns to expand all the columns wide enough to read all of the cell values.

  1. To create a general template that can be used for multiple asset types, delete all data leaving only the header row. To create a template for a single asset type, delete the contents of all those columns except the attributes common to the asset.
  2.  Populate the spreadsheet.
    1. Do not enter any text into the guid column; the contents of the guid column must be blank.
    2. Enter a unique value for each asset in the $aName column.
    3. For sensors and tagged assets, enter the Tag ID in $aAssetTag. The Tag ID is the six-character Group Code followed by the eight-digit Tag ID, with no spaces.
    4. All other columns are optional.
  1. Save as CSV.
  2. Navigate to User Console > Assets > Import Assets.
  3. Browse to and Upload file.
