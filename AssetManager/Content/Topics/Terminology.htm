Information relayed from one or more Zone Managers is used along with user configured policies to track asset location and alerting events.

Asset

An asset is a physical item to be tracked, for which an asset record must be created in . Assets have Types and Attributes, are assigned Tags, and are assigned to or detected in Locations.

Retired and Deleted Assets

Assets that no longer need to be tracked in can be retired or deleted. Assets that are retired from are listed as Retired status as opposed to Active status. Retired assets are not removed from the database and can be Unretired as required. An asset that is deleted will be completely removed from the system and its associated data is deleted.

Asset Type

Asset Types are data definitions used to describe assets that are tracked with . An Asset Type is defined as a category of asset such as IT assets, monitors or people. Within , Asset Types can be arranged in a hierarchy where child asset types will inherit all attributes associated to the parent type. An example of an asset hierarchy would be “IT Asset > Computer > Laptop”. In this example the Laptop asset type is a child of type Computer which is a child of IT Asset. So a Laptop can be described as a Computer which is an IT Asset.

Attribute

An attribute defines a specific property or characteristic of an Asset Type. Attributes are completely user definable. An example of an attribute might be the manufacturer of the asset or the model of the asset.

Calculated Asset Attribute

Calculated asset attributes are attributes that have been derived from the application of a formula to various asset attributes to produce a value. The Administrator can choose to create the formula or can use one of the pre-created calculated asset attributes that are contained within the data schema.

Custom Attribute Type

A custom attribute type is a combination of attributes into a single grouping to represent a specific quality or characteristic. Custom attribute types are associated to asset types just like standard attributes.

System Attribute

System Attributes are similar to regular attributes but are non-configurable except for display color formatting. System attributes hold system information about Asset Types that are inherited from the active RFID tag associated with the asset. Examples of system attributes are motion, tamper, low battery, etc.

Tag

One-way, transmit-only communicators affixed to assets for tracking or monitoring. Tags can be IR-enabled, for use with IR locators. Sensor tags monitor environmental conditions.

Assigned Tag

Assigned tags are asset tags that have been associated or assigned to an asset. If an asset is deleted or retired from the system the associated tag may be placed back in the unassigned tag list.

Detected Tag

Any tag that has successfully transmitted information to a reader appears as Detected until it is imported into the system. Tags move from Detected to Unassigned, then are assigned to assets (or, for sensor tags, defined as assets).

Retired and Deleted Tag

Tags that are associated with retired or deleted assets can be removed from the system or requested to be put back into the Unassigned tag queue for reuse on other assets.

Unassigned Tag

Unassigned tags are tags that are imported into the system but are not currently associated to an asset. All tags that are added to the system are considered to be in the Unassigned state until they are associated with an asset. The Unassigned tag population does not change unless a user explicitly associates a tag to an asset, deletes an Unassigned tag, or imports a detected tag.

Data Schema

The schema of a database system is its structure. The schema defines the tables, the fields in each table, and the relationships between fields and tables. The schema used to track assets in is completely customizable by the Administrator. A schema is comprised of Asset Types, Attributes, and Custom Attributes.

IR Locator

An IR Room or Rack Locator is a beacon that sends out a unique user-defined location code. IR-enabled tags receive this code and bundle it with the information they send to readers.

Reader

RF Code readers such as the M200 or M250, receive information from tags. will be configured with one or more readers to provide real-time tag location and sensor information.

Web Interface

The web interface is the browser-based user interface which is composed of the Administrator and User Consoles. The Administrator Console is designed to manage the system infrastructure and database schema while the User Console manages all aspects of the physical assets.
