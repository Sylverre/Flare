Tag Groups Configuration 2012

Configure for the tag groups of tags you intend to track. The purpose of this step is to enable to “listen” to specific groups of tags and to correctly interpret the data sent from the tags. To do this you can either double-click on the Tag Groups line in the System Configuration box in the dashboard or navigate to Configuration > Tag Groups. The reader configuration screen appears.

  1. Click the New button and select the treatment code associated with the tag group you would like to configure.

All RF Code tags are defined as being members of a specific group, and have a unique tag ID number within that group. When an RF Code reader is configured, it can be supplied with up to 8 (for M200 or R200 Readers) or 16 (for M250 Readers) group code IDs and a corresponding treatment code for each group code. The treatment code instructs the reader how to interpret the payload data for each tag event within that group code. RF Code tags are smart and have the ability to transmit various types of data within its radio frequency beacon such as indicators for motion, panic, tamper, infrared location, humidity, pressure, temperature, and low battery. The group ID is printed on the label of a tag.

TIP: A support page with the most updated list of group/treatment codes is available to help you determine your specific Group Code and Treatment Code: http://www.rfcode.com/codes

  1. Enter the required name, group code, and location update delay setting (start with the default value for this setting and then tune the configuration later if needed).
  1. Click the Save Changes button. The tag group that you have just configured will appear in the list box on the left.
  1. To configure additional group codes, repeat these steps. You will see the additional tag groups appear in the list box to the left.
