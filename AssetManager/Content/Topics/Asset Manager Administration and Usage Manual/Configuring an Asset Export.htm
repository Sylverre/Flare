Configuring an Asset Export File to Populate and then Import

An Asset Export file is a comma-separated values (CSV) file that can be read by any spreadsheet program.

To configure the file in order to import Assets in bulk, perform the following steps:

  1. Highlight all of the cells and then double-click the line between the first two columns to expand all the columns wide enough to read all of the cell values.


  2. Remove all the columns other than Class, Type, guid, Retired, Deletable, $aName, $aAsset Tag:
Class Type guid Retired Deletable $aName $aAsset Tag
typically Entity. Asset Type leave this field empty. will assign a guid on import. For new assets, False. This field describes whether or not a user can delete this asset. Typically True. Assign a unique name for each asset. Enter the unique Tag ID for each tagged asset.

The example below shows the fields required for importing new tags and attributes and presumes a blank template; that is, nothing has been exported in the screenshot below so the guid field is blank.  However, all of the other attribute fields must have values or the import will present errors.
 

  1. Add your assets with attributes and tags to the spreadsheet.

You must leave the guid field blank (empty) when adding new tags and their attributes.  The guid is a unique asset/sensor identifier that is automatically assigned by to each asset or sensor. 

The values you assign to both the $aName and the $aAssetTag attributes must be unique for each asset or sensor.
