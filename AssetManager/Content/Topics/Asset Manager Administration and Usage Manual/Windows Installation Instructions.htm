can be installed using Linux or Windows.

Windows Installation Overview

  1. Create a Microsoft SQL Server or PostgreSQL database and user account with full access (unless installing pilot, in which case you can use the bundled Microsoft SQL Server Express).
  2. Run install EXE from download or CD.
  3. In Select Components window, choose Install with database (unless installing pilot, in which case Install with Microsoft SQL Server Express).
  4. When web interface opens, log in with default account and password to configure .

Detailed instructions for installing in Winows are presented below.

Linux Installation Overview

  1. Create a PostgreSQL database and user account with full access.
  2. Log in to Linux as root.
  3. Download the RPM.
  4. Install the package:  rpm –i rfcode--{version}.x86_64.rpm
  5. Open ports in the firewall: iptables –I INPUT –p tcp --dport {Port #} –j ACCEPT

Detailed instructions for installing in Linux are presented below.

Install in Windows

  1. Run the installer’s executable file from the from the file that you downloaded from the link provided by Support or the provided CD.
  1. When the Setup Wizard starts, click Next to continue.
  2. Read the License Agreement, select the radio button next to I accept the agreement, and then click Next.
  3. Accept the default location for the installation folders and files or choose an alternate path that you would prefer.

If you choose an installation path other than the default, be sure to record this information, which will be required to backup and restore data, and for server migration, and may be helpful should you ever need to contact Support.

  1. Click Next and in the Select Components window, use the drop-down menu and choose the installation option you prefer.

In most cases, choose the first option in the drop-down menu that installs both and Zone Manager. Install only the Zone Manager component to add additional Zone Managers, if required, after you have installed on a server.

  1. On the next window, click Install.
  2. Select Install with Database, or, for a pilot deployment, click OK to install the default database, Microsoft SQL Server Express edition, which is bundled with the installation package.

    Choose to Install with Microsoft SQL Server Express option only if you are installing as a pilot installation.  SQL Server Express is not supported as a full production database for reasons such as scalability, configuration options, and performance.  If you do choose to install the bundled version of SQL Server Express to use as a database initially, be prepared to use it only as a temporary store of data and as a proof of concept.  You will want to start from scratch when you install with a fully functional database when you deploy to a production environment.

    If installing SQL Server Express, you must accept all defaults. Do not change the default Administrator (sa) password, default installation directories, or default database engine settings.  Doing so will prevent from being able to connect to the database for initial setup and initialization.

  3. If installing for a production environment, click OK to install the selected database.
  1. If installing pilot, accept the Microsoft SQL Server Express license agreement and then click Next.
  2. Accept the default installation directories or specify desired directories.
  1. Accept the default Named Instance, Instance ID, and Instance root directory, or specify desired Instance, and then click Next.
  2. Accept the default database engine settings or set as desired and then click Next.
  3. Configure the error reporting settings and then click Next.
  4. Specify the service accounts.
  5. Register your product to receive updates.
