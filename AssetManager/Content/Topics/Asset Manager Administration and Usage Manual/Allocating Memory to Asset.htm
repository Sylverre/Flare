Allocating Memory to

The amount of memory allocated to the software by default may be sufficient for many installations. However, if the number of assets exceeds roughly 5,000, or the calculated field function is heavily used, additional memory resources may be required. Adding physical memory to the server that runs may help, but to fully utilize the free memory, the software must be redefined to allow to use more memory than the default.

To change the amount of memory being used by , edit the system.properties file and change the value of the memory directive. The following example sets the memory footprint to 2 GB or 2048 MB of memory:

wrapper.java.maxmemory=2048
