Integrating with LDAP / Active Directory

The following instructions will help you to configure to connect to an Active Directory or other LDAP server running on Microsoft Windows Server 2008 R2 or Windows Server 2012.

As of v2.8, access to can now be granted based on LDAP Group membership. This means that users that have never logged into can do so without an Admin explicitly creating a User account.  Instead, an Administrator can grant access to to an entire LDAP User Group, for example s.  Thus a new employee to the company can be granted membership to that User Group and therefore be granted login rights based solely on Group membership without any additional administration needed within .   This lightens the load for the system Administrator, especially for installations with a very large User population.
