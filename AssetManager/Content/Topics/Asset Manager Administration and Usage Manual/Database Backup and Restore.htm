Database Backup and Restore without Microsoft SQL Management Studio

It may be necessary to backup and/or restore an SQL database without the ability to use the SQL Management Studio.  In this case, you can use the following SQL commands and run them from the Windows SQL Server. 

To backup an database called “daytona” (which is the default name assigned to the database when it is installed with the SQL Server Express option in the installation wizard) to a file named c:\sqlback.bak, issue the following command on a Windows Command (cmd) prompt on the SQL Server, which may or may not be on the same server as the server application:

SQL Server Express 2012 is bundled with a version of the SQL Management Studio; therefore, you can back up and restore SQL databases, such as the database, using that menu-driven utility.

SQL Express 2008 Syntax:

c:\Users\Administrator>sqlcmd –E –S localhost\RFCASSETMGR –Q “BACKUP DATABASE [daytona] TO DISK=’c:\sqlback.bak'"

 “localhost\RFCASSETMGR”= Installed by default by

“RFCASSETMGR” is the Database “instance” running on the local server (that is, “localhost”). 

The actual database name, running within the SQL instance, is “daytona”
